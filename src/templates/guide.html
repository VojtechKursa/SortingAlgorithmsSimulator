Main page

The main page is available in 2 modes of display:

  • A grid containing all available algorithm variations - Primary display mode for smaller displays
  • A table, containing a list of all algorithms and their properties.

In the table mode, clicking on a row of an algorithm in the table opens a collapsible table underneath, which contains all available variations of the selected algorithm with their respective properties. Clicking on the name of an algorithm or algorithm variation ("?" appears next to the cursor when hovering over the name) opens a description dialog of the selected algorithm or algorithm variation.

Simulator page

The simulator page contains a simulator of a selected algorithm variation. The description dialog can be accessed by clicking the button next to the algorithm variation's name in the header of the page.

Step kinds

There are several kinds of steps you can choose to step through the algorithm in:

  • Code - A step in algorithm's pseudocode
  • Significant - A significant step in the algorithm, like comparing or swapping values or calling methods.
    • This is the default step kind.
  • Algorithmic - A step that contains a possible change in position of values in the internal data structure.

Continuous step controls and step kind selector

The upper part of the page contains control elements for automatically stepping through an algorithm. Use the play control elements to let the simulator "play" the algorithm. The algorithm is stepped through in the steps selected in the step kind selector, which is right next to the play controls. The step kind selector also controls which steps are stepped through when using keyboard to step through the algorithm.

Visualization section

The central part of the simulator is dedicated to the algorithm visualizer and a debugger section.

Visualizer

The algorithm visualizer visualizes the current state of the algorithm. When stepping through the algorithm, it is continuously updated to show the current state in each step.

In the upper part, there are buttons for manually stepping through an algorithm in Algorithmic (A) and Significant (S) steps. The step controls also contain buttons to skip to the beginning or the end of an algorithm. Between the step buttons are step counters, which shows the current step number and a total step count for the associated step. The step count is only known after reaching the end of an algorithm. Before that, the total step count simply shows "?".

Next to the step controls are renderer settings, where the current renderer can be changed and animations can be turned on or off.

The bottom includes a space for a textual description of the current step.

Debugger

The debugger section imitates a debugger and can be used to closely follow the algorithms's code. It features a display of the selected algorithm's pseudocode with highlighting of the current instruction. There is also a table of variables which shows the variables of an algorithm in the current scope.

Some algorithms also have a call stack tracker in this section. The call stack tracker shows the current state of the algorithm's call stack, represented by names of functions at the respective levels of the call stack, along with variables and their values at the respective levels.

At the top of the debugger section is a step control element, which controls steps of the Code (C) kind, along with associated step counter.

The whole debugger section can be hidden. The variable and call stack displays can also be hidden independently.

Input and reset buttons

Below the visualization section are buttons for resetting the algorithm and setting an input of the algorithm. The input button opens a dialog that allows the user to set the input values for the simulator.
